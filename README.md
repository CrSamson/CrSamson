Design a GitHub profile README hero section for a developer named Cristian Samson.

Identity:
- Name: Cristian Samson
- Title: Applied AI Developer
- Location: Montréal
- Motto: "Stay humble. Keep learning. Build things people actually use."

Visual direction: technical and calm. Serious-engineer aesthetic — not playful, not corporate. Think GitHub's own dark theme but more refined.

Color palette (use these exact hex values):
- Background: #0B1220 (deep navy, near-black with blue undertone)
- Secondary surface: #1A2332 (one shade lighter, for any layered elements)
- Primary text: #E2E8F0 (off-white)
- Muted text: #94A3B8 (slate, for secondary info)
- Accent: #22D3EE (electric cyan — use sparingly, one or two moments only)

Layout:
- Wide hero banner at top (roughly 1200×300 ratio) containing the name, title, location, and motto
- The accent cyan should appear as a single deliberate element — for example, a thin vertical bar to the left of the text, or one underline. Not multiple cyan elements.
- Generous whitespace around text. The calm comes from breathing room, not from added decoration.
- Below the banner: a centered row of two contact buttons (LinkedIn, Email) styled as flat pill badges
- Below that: section headers like "What I Ship", "Currently Building", "Now Learning", "Highlighted Work", "Tech Stack", "Education"
- Each section minimal — no heavy dividers, no boxes, no card UI

Typography:
- Sans-serif throughout (system stack: -apple-system, Segoe UI, Roboto)
- Name: large, weight 700
- Title: medium size, weight 500, muted color
- Motto: smaller, italic, more muted

Do NOT include:
- GitHub stats cards, streak cards, trophy widgets
- Multiple accent colors or gradients
- Decorative illustrations, emojis as visual elements (small inline emojis in section headers are okay)
- Animated elements
- Marketing language or gradients

The goal is a hero section that signals seriousness and craft within five seconds — the kind of profile a senior engineer or hiring manager respects on first look.
